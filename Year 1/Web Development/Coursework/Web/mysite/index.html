Week 6: Introduction to responsive web design;

Published on October 31st, 2017.

Week 6 has been a real head scratcher for me and from what I have gathered many other students. You need to format your CSS so that when your webpage is loaded up on different devices and size screen it still looks how you want it to and is functional; this is called responsiveness, i.e. the webpages layout remains the same on whatever size screen. For the life of me I cannot seem to get it to work. The first thing you need to do is sketch out how you want your webpage to look so that when you are creating the CSS you know what to do to get the template you desire via the correct use of different elements and content., essentially bringing in the 5 P's (Proper Planning Prevents Poor Performance).

Once you have all this, you need to add the "meta tag" to the "head" of the document; this is to make the mobile version of the browser behave like the desktop browser and display the full web page size. Furthermore, once that is all done and you have ironed out the nicks I needed to adjust the HTLM a little further so everything stayed how I wanted to. Now here is where my brain exploded a little and it took me a while to do, you need to re-arrange the CSS completely; i.e. you need to take every bit of code and put it in it's respective place as up to now I just put one bit after the other as I went along. Which is fine, but poor practice and now I definitely will use the correct layout right off the bat to save myself some trouble. Next I styled the small screen layout (for mobile devices), this way again it keeps the template you have designed. In addition, you will also need to do all this for wider screens.

Once I had done all this I really thought right surely now the webpage will behave itself across all platforms and size screens. Boy I was wrong, I still needed to make any tables I had and images responsive. All in all this week has been the toughest so far and a real struggle for me.

Week 7: RWD2 Flexible Media;

Published on November 14th, 2017.

After the last 2 weeks experience I have a much better understanding of what exactly it is I am doing and how the changes are affecting my webpage. For example, the teacher recommended we float the references on the left of the screen. I originally didn't like the look of this or understand why I am was doing this. But after quite a lot of playing around and looking at online guides I have finally understood and implemented this function in a way I want and desire. Especially with me wanting to put the week links on the right hand side of the screen which in its own was a real pain. Turns out that making meida (tables and images in my case) is actually relatively straight forward. You simply set the width to 100% of the screen width and then make set the width to the size of whatever screen the webpage is displayed on. You essentially do this by setting boundaries, for example a screen with a minimum width of 55em and then another statement for screens bigger than 55em. 1em equals 16 pixels, so 55 em will give you a screen of 880 pixels, this adds up to a screen of 11.73 inches or 30cm.
